ğŸ“‹ Overview

The RBAC Manager is a comprehensive Python tool designed to automate the extraction and generation of RBAC (Role-Based Access Control) resources for OLMv1 operator deployments. It bridges the gap between operator bundle metadata and production-ready Kubernetes RBAC manifests.

Key Capabilities

ğŸ” Catalog Discovery

List and query OpenShift ClusterCatalogs to discover available operators, channels, and versions.

ğŸ“¦ Bundle Analysis

Extract comprehensive metadata from operator bundle images using the OPM tool.

ğŸ›¡ï¸ Smart RBAC Generation

Auto-generate secure RBAC resources with intelligent permission deduplication and optimization.

âš™ï¸ Helm Integration

Generate Helm values files compatible with the generic OLMv1 Helm chart.

ğŸ” Security Best Practices

Implements OLMv1 security patterns with post-installation hardening guidance.

ğŸ“ Configuration Management

Generate and reuse configuration files for consistent deployments across environments.

Design Patterns

Strategy Pattern

Used in RBAC generation to handle different permission scenarios:

  • BothPermissionsStrategy - Operators with cluster + namespace permissions
  • ClusterOnlyStrategy - Cluster-scoped operators
  • NamespaceAsClusterStrategy - Namespace permissions treated as cluster
  • NoPermissionsStrategy - Minimal operators

Factory Pattern

Strategy selection based on permission analysis, template creation for manifests and Helm values.

Template Method Pattern

Base generator class defines common RBAC generation workflow, subclasses implement format-specific details.

Flag Validation Rules

âš ï¸ Important: The following flag combinations are mutually exclusive:
  • --openshift-namespace and --helm cannot be used together
  • --openshift-namespace is only valid for YAML generation (when --helm is omitted)
  • For Helm generation, use --helm without --openshift-namespace

ğŸ”„ Common Workflows

Workflow 1: Configuration-Based Deployment (Recommended)

Step 1: List available catalogs 
python3 rbac-manager.py list-catalogs --openshift-url https://api.cluster.example.com:6443 --openshift-token sha256~token
Step 2: Generate configuration from catalogd 
python3 rbac-manager.py catalogd --to-config \
  --catalog-name openshift-community-operators \
  --package argocd-operator --channel alpha --version 0.8.0 \
  --openshift-url https://api.cluster.example.com:6443 \
  --openshift-token sha256~token --skip-tls \
  --output ./config
Step 3: Extract RBAC using configuration 
python3 rbac-manager.py opm --config ./config/argocd-operator-rbac-config.yaml
Step 4: Deploy RBAC resources 
kubectl apply -f argocd-operator-serviceaccount-*.yaml
kubectl apply -f argocd-operator-clusterrole-*.yaml
kubectl apply -f argocd-operator-clusterrolebinding-*.yaml

Workflow 2: Direct Bundle Extraction

Step 1: Extract RBAC directly from bundle image 
python3 rbac-manager.py opm \
  --image quay.io/operatorhubio/argocd-operator:v0.12.0 \
  --namespace argocd-operator \
  --output ./rbac-files
Step 2: Review generated files 
ls -la ./rbac-files/
Step 3: Apply to cluster 
kubectl apply -f ./rbac-files/

Workflow 3: Helm Values Generation

Step 1: Generate Helm values 
python3 rbac-manager.py opm \
  --image quay.io/operatorhubio/argocd-operator:v0.12.0 \
  --helm \
  --output ./helm-values
Step 2: Review and customize values 
cat ./helm-values/argocd-operator-values.yaml
Step 3: Deploy with Helm 
helm install argocd-operator ./helm-chart \
  -f ./helm-values/argocd-operator-values.yaml
